Path generators replace the default straight-line movement with algorithmic trajectories. Each path is a separate package that extends the `@tsparticles/plugin-move` system. You enable a path on a particle group by setting `particles.move.path.generator` to the generator's registered name.

## How paths work

Path generators are registered via `addPathGenerator` on a `MoveEngine` — an extension of the base engine provided by `@tsparticles/plugin-move`. Every path package calls `ensureBaseMoverLoaded` to make sure the move plugin is present, then registers its generator.

```typescript
import { tsParticles } from "@tsparticles/engine";
import { loadPerlinNoisePath } from "@tsparticles/path-perlin-noise";

await loadPerlinNoisePath(tsParticles);

await tsParticles.load({
  id: "canvas",
  options: {
    particles: {
      move: {
        enable: true,
        path: {
          enable: true,
          generator: "perlinNoise",
        },
      },
    },
  },
});
```

<Info>
  All path packages depend on `@tsparticles/plugin-move` as a peer dependency. Make sure it is installed, or use a bundle that includes it.
</Info>

## Configuring the path option

The `particles.move.path` block controls path behavior:

```json
{
  "particles": {
    "move": {
      "enable": true,
      "path": {
        "enable": true,
        "generator": "<generatorName>",
        "options": {
          // generator-specific options go here
        },
        "delay": {
          "value": 0
        },
        "clamp": true
      }
    }
  }
}
```

| Field | Description |
|-------|-------------|
| `enable` | Activates path-based movement |
| `generator` | The registered generator name |
| `options` | Generator-specific configuration |
| `delay.value` | Seconds to wait before path movement starts |
| `clamp` | Whether to clamp path vector values |

## Available paths

### branches

**Package:** `@tsparticles/path-branches` · **Function:** `loadBranchesPath` · **Generator:** `"branchesPathGenerator"`

Simulates organic branching growth — particles move in segments and occasionally make sharp directional changes. Configurable branch probability, segment length, and optional inertia-based smooth interpolation.

```typescript
import { loadBranchesPath } from "@tsparticles/path-branches";
await loadBranchesPath(engine);
```

```json
{
  "particles": {
    "move": {
      "path": {
        "enable": true,
        "generator": "branchesPathGenerator",
        "options": {
          "branchChance": 0.1,
          "segmentLength": 20,
          "maxAngle": 0.5,
          "speedVariation": 0.2,
          "inertia": { "enable": true, "factor": 0.05 }
        }
      }
    }
  }
}
```

### grid

**Package:** `@tsparticles/path-grid` · **Function:** `loadGridPath` · **Generator:** `"gridPathGenerator"`

Moves particles along a grid graph. The grid can be auto-generated as a random maze, or you can supply an explicit adjacency graph. Particles traverse the graph cells, producing structured, maze-like motion.

```typescript
import { loadGridPath } from "@tsparticles/path-grid";
await loadGridPath(engine);
```

```json
{
  "particles": {
    "move": {
      "path": {
        "enable": true,
        "generator": "gridPathGenerator",
        "options": {
          "cellSize": 50,
          "autoMaze": true
        }
      }
    }
  }
}
```

| Option | Description |
|--------|-------------|
| `cellSize` | Size in pixels of each grid cell |
| `autoMaze` | Generate a random maze automatically |
| `graph` | Explicit adjacency graph (`"x,y"` → `[allowed directions]`) |

### perlinNoise

**Package:** `@tsparticles/path-perlin-noise` · **Function:** `loadPerlinNoisePath` · **Generator:** `"perlinNoise"`

Samples a Perlin noise field to compute smooth, continuous velocity vectors. Particles flow along coherent stream-lines that change slowly over time. This is the most commonly used path generator for organic, fluid-like effects.

**Peer dependencies:** `@tsparticles/noise-field`, `@tsparticles/perlin-noise`, `@tsparticles/plugin-move`

      "speed": 2,
      "path": {
        "enable": true,
        "generator": "perlinNoise"

    }
  }
}

### svg

**Package:** `@tsparticles/path-svg` · **Function:** `loadSVGPath` · **Generator:** `"svgPathGenerator"`

Traces particle motion along the path data of an inline SVG element in the DOM. Particles follow the exact curves defined by the SVG `<path>` `d` attribute, making it possible to animate text outlines, logos, or any arbitrary vector shape.

```typescript
import { loadSVGPath } from "@tsparticles/path-svg";
await loadSVGPath(engine);
```

```json
{
  "particles": {
    "move": {
      "path": {
        "enable": true,
        "generator": "svgPathGenerator",
        "options": {
          "element": "#svgPath"
        }
      }
    }
  }
}
```
