tsParticles separates visual features into two distinct systems that both run every animation frame:

- **Effects** implement `IEffectDrawer` and add canvas-level rendering — shadow blurs, trailing lines, bubble highlights, CSS filter functions, and more. They wrap the particle's shape draw call via `drawBefore` and `drawAfter`.
- **Updaters** implement `IParticleUpdater` and modify particle state — opacity, size, rotation angle, color, lifetime, orbit position, and so on. They run `update()` every frame and can also contribute color styles and canvas transform values.

## Effects

### bubble

**Package:** `@tsparticles/effect-bubble` · **Function:** `loadBubbleEffect` · **Effect name:** `"bubble"`

Adds a white highlight arc in the upper-left of each particle after the shape is drawn, giving a soap-bubble or glass-ball appearance.

```typescript
import { loadBubbleEffect } from "@tsparticles/effect-bubble";
await loadBubbleEffect(engine);
```

```json
{
  "particles": {
    "shape": { "type": "circle" },
    "effect": {
      "type": "bubble"
    }
  }
}
```

The bubble is drawn by `BubbleDrawer.drawAfter`, which places a filled arc at `radius / 3` offset from the particle center. No per-particle configuration is required.

### trail

**Package:** `@tsparticles/effect-trail` · **Function:** `loadTrailEffect` · **Effect name:** `"trail"`

Renders a fading trail behind each particle. The trail is drawn into the canvas using the particle's current position history, creating a motion-blur comet effect.

```typescript
import { loadTrailEffect } from "@tsparticles/effect-trail";
await loadTrailEffect(engine);
```

```json
{
  "particles": {
    "effect": {
      "type": "trail",
      "options": {
        "trail": {
          "length": 20,
          "minWidth": 1
        }
      }
    }
  }
}
```

<Note>
  The `@tsparticles/fireworks` bundle includes the trail effect by default. If you are building a fireworks animation from scratch, install `@tsparticles/effect-trail` alongside the emitters plugin.
</Note>

### shadow

**Package:** `@tsparticles/effect-shadow` · **Function:** `loadShadowEffect` · **Effect name:** `"shadow"`

Applies a canvas `shadowBlur` / `shadowColor` / `shadowOffset` to each particle before drawing. The shadow color, blur radius, and x/y offset are configurable per particle.

```typescript
import { loadShadowEffect } from "@tsparticles/effect-shadow";
await loadShadowEffect(engine);
```

```json
{
  "particles": {
    "effect": {
      "type": "shadow",
      "options": {
        "shadow": {
          "color": { "value": "#ff0000" },
          "blur": 10,
          "offset": { "x": 0, "y": 0 }
        }
      }
    }
  }
}
```

The `ShadowDrawer` saves and restores the canvas context around each particle draw call, setting `context.shadowBlur`, `context.shadowColor`, `context.shadowOffsetX`, and `context.shadowOffsetY`.

### filter

**Package:** `@tsparticles/effect-filter` · **Function:** `loadFilterEffect` · **Effect name:** `"filter"`

Applies a CSS `filter()` function to each particle using the canvas context's `filter` property. All standard CSS filter functions are supported.

```typescript
import { loadFilterEffect } from "@tsparticles/effect-filter";
await loadFilterEffect(engine);
```

```json
{
  "particles": {
    "effect": {
      "type": "filter",
      "options": {
        "filter": {
          "blur": 2,
          "brightness": 1.2,
          "contrast": 1,
          "grayscale": 0,
          "hueRotate": 0,
          "invert": 0,
          "opacity": 1,
          "saturate": 1,
          "sepia": 0,
          "dropShadow": "0 0 4px #fff",
          "url": ""
        }
      }
    }
  }
}
```

The `IFilterData` interface:

```typescript
export interface IFilterData extends IShapeValues {
  blur?: number | string;
  brightness?: number;
  contrast?: number;
  dropShadow?: string;
  grayscale?: number;
  hueRotate?: number | string;
  invert?: number;
  opacity?: number;
  saturate?: number;
  sepia?: number;
  url?: string;
}
```

## How effects interact with the rendering pipeline

The canvas rendering sequence for each particle is:

<Steps>
  <Step title="effect.drawBefore">
    The effect drawer's `drawBefore` method runs. This is where shadow effects save the canvas context and set `context.shadowBlur` / `context.shadowColor`.
  </Step>
  <Step title="Shape draw">
    The shape drawer's `draw` method renders the particle geometry into the canvas context.
  </Step>
  <Step title="effect.drawAfter">
    The effect drawer's `drawAfter` method runs. This is where the bubble highlight arc is added on top of the shape.
  </Step>
</Steps>

Effects are selected via `particles.effect.type`. Per-particle effect data is stored under `particles.effect.options.<effectName>` and is accessible in the drawer via `particle.effectData`.

## Particle updaters

Updaters implement `IParticleUpdater` and run every frame. Each updater is registered under a key name and applies to all particles in a container. Most updaters read per-particle options from the configuration block that matches their feature area.

### opacity

**Package:** `@tsparticles/updater-opacity` · **Function:** `loadOpacityUpdater` · **Key:** `"opacity"`

Animates particle opacity over time. Reads `particles.opacity.animation` to drive a looping or one-shot fade.

```typescript
import { loadOpacityUpdater } from "@tsparticles/updater-opacity";
await loadOpacityUpdater(engine);
```

```json
{
  "particles": {
    "opacity": {
      "value": { "min": 0.1, "max": 1 },
      "animation": {
        "enable": true,
        "speed": 1,
        "sync": false
      }
    }
  }
}
```

### life

**Package:** `@tsparticles/updater-life` · **Function:** `loadLifeUpdater` · **Key:** `"life"`

Manages particle lifetime — each particle lives for a configurable duration, then is destroyed and optionally re-spawned a fixed number of times. Supports a spawn delay and `sync` to align lifetimes across particles.

```typescript
import { loadLifeUpdater } from "@tsparticles/updater-life";
await loadLifeUpdater(engine);
```

```json
{
  "particles": {
    "life": {
      "count": 3,
      "delay": {
        "value": 0.1,
        "sync": false
      },
      "duration": {
        "value": { "min": 1, "max": 3 },
        "sync": false
      }
    }
  }
}
```

Setting `count` to `0` gives particles infinite lives. Setting `duration.value` to `0` or less makes them live indefinitely.

### out-modes

**Package:** `@tsparticles/updater-out-modes` · **Function:** `loadOutModesUpdater` · **Key:** `"outModes"`

Controls what happens when a particle reaches the edge of the canvas. Modes include bouncing off the boundary, wrapping to the opposite side, being destroyed, or being deflected. This updater is included in almost all bundles.

```typescript
import { loadOutModesUpdater } from "@tsparticles/updater-out-modes";
await loadOutModesUpdater(engine);
```

```json
{
  "particles": {
    "move": {
      "outModes": {
        "default": "bounce",
        "top": "out",
        "bottom": "destroy"
      }
    }
  }
}
```

Available mode values: `"out"`, `"bounce"`, `"bounceHorizontal"`, `"bounceVertical"`, `"destroy"`, `"none"`.

## Writing a custom updater

Implement `IParticleUpdater` from `@tsparticles/engine` and register it with `engine.pluginManager.addParticleUpdater`:

```typescript
import { type Engine, type IDelta, type IParticleUpdater, type Particle } from "@tsparticles/engine";

class PulseUpdater implements IParticleUpdater {
  init(particle: Particle): void {
    // set up any per-particle state here
  }

  isEnabled(particle: Particle): boolean {
    return !particle.destroyed;
  }

  update(particle: Particle, delta: IDelta): void {
    // modify particle state every frame
    // delta.value is the time elapsed in milliseconds
  }
}

export async function loadPulseUpdater(engine: Engine): Promise<void> {
  await engine.pluginManager.register(e => {
    e.pluginManager.addParticleUpdater("pulse", async () => new PulseUpdater());
  });
}
```
