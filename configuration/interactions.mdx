tsParticles supports two categories of interactions:

- **External interactions** — triggered by the mouse cursor or pointer events (hover, click, scroll).
- **Particle interactions** — behaviors that occur between particles themselves (links, collisions, attraction, repulsion).

Each interaction is provided as a separate npm package and must be loaded into the engine before it can be used. The `tsparticles` full bundle and `@tsparticles/slim` include the most common interactions by default.

<Info>
  Interaction options are nested under the plugin's key inside `interactivity` (for external) or `particles` (for particle-to-particle). The exact nesting paths are shown per interaction below.
</Info>

---

## External interactions

External interactions respond to mouse/pointer events. They are configured under `interactivity.events` (to enable the trigger) and under `interactivity.modes` (to configure the behavior).

```json
{
  "interactivity": {
    "detectsOn": "window",
    "events": {
      "onHover": { "enable": true, "mode": "repulse" },
      "onClick": { "enable": true, "mode": "push" }
    },
    "modes": {
      "repulse": { "distance": 100, "duration": 0.4 }
    }
  }
}
```

<Note>
  Multiple modes can be active simultaneously by passing an array: `"mode": ["repulse", "bubble"]`.
</Note>

### Attract

**Package:** `@tsparticles/interaction-external-attract`

Pulls nearby particles toward the cursor while the pointer hovers.

<AccordionGroup>
  <Accordion title="Configuration options">
    <ParamField body="interactivity.modes.attract.distance" type="number" default="200">
      Radius in pixels around the cursor within which particles are attracted.
    </ParamField>

    <ParamField body="interactivity.modes.attract.duration" type="number" default="0.4">
      Duration in seconds of the attraction effect after the cursor moves away.
    </ParamField>

    <ParamField body="interactivity.modes.attract.easing" type="string" default="easeOutQuad">
      Easing function applied to the attraction force. Common values: `"easeOutQuad"`, `"easeInQuad"`, `"linear"`.
    </ParamField>

    <ParamField body="interactivity.modes.attract.factor" type="number" default="1">
      Multiplier applied to the attraction force.
    </ParamField>

    <ParamField body="interactivity.modes.attract.maxSpeed" type="number" default="50">
      Maximum speed at which particles are pulled toward the cursor.
    </ParamField>

    <ParamField body="interactivity.modes.attract.speed" type="number" default="1">
      Base speed of the attraction movement.
    </ParamField>
  </Accordion>
  <Accordion title="Example">
    ```json
    {
      "interactivity": {
        "events": { "onHover": { "enable": true, "mode": "attract" } },
        "modes": {
          "attract": {
            "distance": 200,
            "duration": 0.4,
            "factor": 5,
            "maxSpeed": 50
          }
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Bubble

**Package:** `@tsparticles/interaction-external-bubble`

Particles near the cursor temporarily change size, opacity, or color, creating a bubble effect.

<AccordionGroup>
  <Accordion title="Configuration options">
    <ParamField body="interactivity.modes.bubble.distance" type="number" default="400">
      Radius in pixels around the cursor in which the bubble effect is applied.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.duration" type="number" default="2">
      Duration in seconds to transition in/out of the bubble state.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.size" type="number">
      Target size (radius in pixels) for particles inside the bubble radius.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.opacity" type="number">
      Target opacity for particles inside the bubble radius.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.color" type="string | object | array">
      Target color for particles inside the bubble radius.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.mix" type="boolean" default="false">
      When `true`, mixes the bubble color with the particle's original color instead of replacing it.
    </ParamField>

    <ParamField body="interactivity.modes.bubble.divs" type="object | array">
      Applies the bubble effect to specific DOM elements instead of the cursor position. See the divs configuration for details.
    </ParamField>
  </Accordion>
  <Accordion title="Example">
    ```json
    {
      "interactivity": {
        "events": { "onHover": { "enable": true, "mode": "bubble" } },
        "modes": {
          "bubble": {
            "distance": 250,
            "duration": 0.5,
            "size": 10,
            "opacity": 1
          }
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Pause

**Package:** `@tsparticles/interaction-external-pause`

Pauses the entire animation while the cursor hovers over the canvas.

No additional mode options are required. Enable it via the hover event:

```json
{
  "interactivity": {
    "events": { "onHover": { "enable": true, "mode": "pause" } }
  }
}
```

### Trail

**Package:** `@tsparticles/interaction-external-trail`

Emits particles along the path of the cursor as it moves, leaving a trail.

<AccordionGroup>

    <ParamField body="interactivity.modes.trail.quantity" type="number" default="1">

    </ParamField>

      When `true`, stops emitting particles when the cursor is stationary.
    </ParamField>

    </ParamField>
  </Accordion>
  <Accordion title="Example">
    ```json
    {
      "interactivity": {
        "events": { "onHover": { "enable": true, "mode": "trail" } },
        "modes": {
          "trail": {
            "delay": 0.005,
            "pauseOnStop": true,
            "quantity": 5,
            "particles": {
              "size": { "value": 3 },
              "move": { "speed": 1, "outModes": "destroy" }
            }
          }
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Particle-to-particle interactions

These interactions govern behaviors between particles in the same container.

### Links

**Package:** `@tsparticles/interaction-particles-links`

Draws lines between particles that are close to each other, creating a network mesh effect.

<AccordionGroup>
  <Accordion title="Configuration options">
    <ParamField body="particles.links.enable" type="boolean" default="false">
      Master switch for the links interaction.
    </ParamField>

    <ParamField body="particles.links.distance" type="number" default="150">
      Maximum distance in pixels between two particles for a link line to appear.
    </ParamField>

    <ParamField body="particles.links.color" type="string | object" default="#ffffff">
      Color of the link lines. Use `"random"` for random colors per link. Accepts a CSS color string or color object.
    </ParamField>

    <ParamField body="particles.links.opacity" type="number" default="0.4">
      Maximum opacity of link lines. Lines fade with distance.
    </ParamField>

    <ParamField body="particles.links.width" type="number" default="1">
      Width of the link lines in pixels.
    </ParamField>

    <ParamField body="particles.links.blink" type="boolean" default="false">
      When `true` and `color` is `"random"`, the link color changes every frame, creating a blinking effect.
    </ParamField>

    <ParamField body="particles.links.consent" type="boolean" default="false">
      When `true` and `color` is `"random"`, all links use the same single random color.
    </ParamField>

    <ParamField body="particles.links.frequency" type="number" default="1">
      Probability that a link is drawn between two eligible particles. `1` = always draw, `0.5` = draw half the time.
    </ParamField>

    <ParamField body="particles.links.id" type="string">
      When set, only particles sharing the same `id` will link with each other. Useful for separating link groups.
    </ParamField>

    <ParamField body="particles.links.warp" type="boolean" default="false">
      When `true`, links wrap across canvas edges — particles near opposite edges will be linked as if the canvas is a torus.
    </ParamField>

    <ParamField body="particles.links.shadow" type="object">
      Draws a glow shadow behind each link line. Heavy on performance; use sparingly.

      <Expandable title="shadow properties">
        <ParamField body="particles.links.shadow.enable" type="boolean" default="false">
          Enables shadow rendering on links.
        </ParamField>

        <ParamField body="particles.links.shadow.color" type="string" default="#000000">
          Shadow color.
        </ParamField>

        <ParamField body="particles.links.shadow.blur" type="number" default="5">
          Shadow blur radius in pixels.
        </ParamField>
      </Expandable>
    </ParamField>

    <ParamField body="particles.links.triangles" type="object">
      Fills the triangular areas formed between three mutually-linked particles.

      <Expandable title="triangles properties">
        <ParamField body="particles.links.triangles.enable" type="boolean" default="false">
          Enables triangle fill rendering.
        </ParamField>

        <ParamField body="particles.links.triangles.color" type="string | object">
          Fill color for the triangles.
        </ParamField>

        <ParamField body="particles.links.triangles.opacity" type="number">
          Opacity of the triangle fill.
        </ParamField>

        <ParamField body="particles.links.triangles.frequency" type="number" default="1">
          Probability that a valid triangle is filled.
        </ParamField>
      </Expandable>
    </ParamField>
  </Accordion>
  <Accordion title="Example">
    ```json
    {
      "particles": {
        "number": { "value": 80 },
        "links": {
          "enable": true,
          "distance": 150,
          "color": "#ffffff",
          "opacity": 0.4,
          "width": 1
        },
        "move": { "enable": true, "speed": 2 }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

### Collisions

**Package:** `@tsparticles/interaction-particles-collisions`

Detects and responds to physical collisions between particles.

<AccordionGroup>
  <Accordion title="Configuration options">
    <ParamField body="particles.collisions.enable" type="boolean" default="false">
      Enables collision detection between particles.
    </ParamField>

    <ParamField body="particles.collisions.mode" type="string" default="bounce">
      Collision response mode:
      - `"bounce"` — particles bounce off each other.
      - `"absorb"` — one particle absorbs the other, growing larger.
      - `"destroy"` — the smaller particle is destroyed.
    </ParamField>

    <ParamField body="particles.collisions.maxSpeed" type="number | RangeValue" default="50">
      Maximum speed cap applied after a collision.
    </ParamField>

    <ParamField body="particles.collisions.absorb.speed" type="number" default="2">
      Speed at which the absorbing particle grows (in absorb mode).
    </ParamField>

    <ParamField body="particles.collisions.bounce" type="object">
      Energy retention for bounces, same shape as `particles.bounce`.

      <Expandable title="bounce properties">
        <ParamField body="particles.collisions.bounce.horizontal.value" type="number | RangeValue" default="1">
          Horizontal energy factor on collision bounce.
        </ParamField>

        <ParamField body="particles.collisions.bounce.vertical.value" type="number | RangeValue" default="1">
          Vertical energy factor on collision bounce.
        </ParamField>
      </Expandable>
    </ParamField>

    <ParamField body="particles.collisions.overlap" type="object">
      Controls how overlapping particles are handled.

      <Expandable title="overlap properties">
        <ParamField body="particles.collisions.overlap.enable" type="boolean" default="true">
          Allows particles to overlap when they first appear. Disabling forces particles to be placed without overlap.
        </ParamField>

        <ParamField body="particles.collisions.overlap.retries" type="number" default="0">
          Number of placement retry attempts to find a non-overlapping position.
        </ParamField>
      </Expandable>
    </ParamField>
  </Accordion>
  <Accordion title="Example">
    ```json
    {
      "particles": {
        "number": { "value": 30 },
        "size": { "value": { "min": 5, "max": 20 } },
        "collisions": {
          "enable": true,
          "mode": "bounce",
          "maxSpeed": 30
        },
        "move": { "enable": true, "speed": 3 }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## Combined interactions example

The following configuration shows a common setup with multiple interactions active at once:

```json
{
  "interactivity": {
    "detectsOn": "window",
    "events": {
      "onHover": { "enable": true, "mode": ["grab", "bubble"] },
      "onClick": { "enable": true, "mode": "push" }
    },
    "modes": {
      "grab": {
        "distance": 300,
        "links": { "opacity": 0.7, "color": "#ffffff" }
      },
      "bubble": {
        "distance": 200,
        "size": 8,
        "opacity": 1,
        "duration": 0.3
      },
      "push": { "quantity": 3 }
    }
  },
  "particles": {
    "number": { "value": 80 },
    "links": {
      "enable": true,
      "distance": 150,
```
