## Choose your installation method

<Tabs>
  <Tab title="CDN">
    Add a script tag to your HTML — no build step required.

    ```html index.html
    <!DOCTYPE html>
    <html>
      <head><title>tsParticles</title></head>
      <body>
        <div id="tsparticles"></div>

        <!-- Full bundle via jsDelivr -->
        <script src="https://cdn.jsdelivr.net/npm/tsparticles/tsparticles.bundle.min.js"></script>
        <script>
          tsParticles.load({
            id: "tsparticles",
            options: {
              particles: {
                number: { value: 80 },
                color: { value: "#895ea6" },
                links: { enable: true, color: "#895ea6", opacity: 0.4 },
                move: { enable: true, speed: 2 },
                opacity: { value: 0.5 },
                size: { value: { min: 1, max: 3 } }
              },
              interactivity: {
                events: {
                  onHover: { enable: true, mode: "repulse" }
                }
              }
            }
          });
        </script>
      </body>
    </html>
    ```
  </Tab>
  <Tab title="npm / yarn / pnpm">
    <Steps>
      <Step title="Install a bundle">
        Pick the bundle that fits your needs. `tsparticles` (the full bundle) is a good default.

        <CodeGroup>
          ```bash npm
          npm install tsparticles
          ```
          ```bash yarn
          yarn add tsparticles
          ```
          ```bash pnpm
          pnpm add tsparticles
          ```
        </CodeGroup>

        <Tip>
          Need something lighter? Use `@tsparticles/slim` for interactions without paths/effects, or `@tsparticles/basic` for bare-minimum moving dots. See [Bundles](/bundles) for a full comparison.
        </Tip>
      </Step>
      <Step title="Load the engine and initialize">
        Import the bundle loader and call it with the `tsParticles` singleton before calling `load`.

        ```typescript app.ts
        import { tsParticles } from "@tsparticles/engine";
        import { loadFull } from "tsparticles";

        // Load all features from the full bundle
        await loadFull(tsParticles);

        // Create a particle container
        const container = await tsParticles.load({
          id: "tsparticles",
          options: {
            particles: {
              number: { value: 80 },
              color: { value: "#895ea6" },
              links: { enable: true, color: "#895ea6", opacity: 0.4 },
              move: { enable: true, speed: 2 },
              opacity: { value: 0.5 },
              size: { value: { min: 1, max: 3 } }
            },
            interactivity: {
              events: {
                onHover: { enable: true, mode: "repulse" }
              }
            }
          }
        });
        ```
      </Step>
      <Step title="Add the container element">
        Add a `div` with the matching `id` to your HTML. tsParticles creates a canvas inside it automatically.

        ```html
        <div id="tsparticles" style="width: 100%; height: 100vh;"></div>
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Control the animation

The `load()` call returns a `Container` instance you can use to control playback:

```typescript
const container = await tsParticles.load({ id: "tsparticles", options: { /* ... */ } });

container.pause();   // freeze animation
container.play();    // resume animation
container.refresh(); // reload with current options
container.destroy(); // remove canvas and clean up
```

To access a previously loaded container by index:

```typescript
const container = tsParticles.item(0); // first loaded instance
container?.pause();
```

## Load options from a JSON file

Instead of inline options, you can point to a JSON file:

```typescript
await tsParticles.load({
  id: "tsparticles",
  url: "/particles.json"
});
```
