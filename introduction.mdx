tsParticles is a feature-rich, open-source library built on top of the HTML5 Canvas API. It lets you add animated particle effects — from simple floating dots and connected networks to full confetti bursts and fireworks displays — to any web project. The source language is TypeScript, but all distributed files are plain JavaScript, so no TypeScript tooling is required to use it.

## Key features

- **Dependency-free** — the engine has no runtime dependencies
- **Browser-ready** — works via CDN `<script>` tags or as an ES module
- **Highly configurable** — control every aspect of particle behaviour through a single JSON options object
- **Plugin architecture** — shapes, paths, updaters, interactions, and effects are all separate packages that you load on demand
- **Bundle system** — pre-assembled bundles let you pick a feature set without manually wiring up individual plugins
- **Framework integrations** — official wrapper components for React, Vue 2/3, Angular, Svelte, Preact, Inferno, SolidJS, Riot.js, Astro, Ember.js, jQuery, and Web Components

## The engine

The `@tsparticles/engine` package is the core of tsParticles. It exposes a singleton `tsParticles` instance (available as `globalThis.tsParticles` in the browser) that manages all particle containers on the page. The engine itself ships with no built-in shapes, interactions, or updaters — these are added by loading plugins or bundles.

```typescript
import { tsParticles } from "@tsparticles/engine";
```

The engine is designed to be extended. Every visual feature — circle shapes, colour updaters, mouse interactions, emitters — is implemented as an independent plugin package that registers itself with the engine.

## Plugin architecture

Plugins fall into several categories:

| Category | Description | Example packages |
|----------|-------------|-----------------|
| **Shapes** | Define particle geometry | `@tsparticles/shape-circle`, `@tsparticles/shape-star` |
| **Updaters** | Modify particle properties each frame | `@tsparticles/updater-opacity`, `@tsparticles/updater-rotate` |
| **Interactions** | Handle pointer and particle-to-particle events | `@tsparticles/interaction-external-repulse`, `@tsparticles/interaction-particles-links` |
| **Effects** | Render canvas-level visual effects | `@tsparticles/effect-trail`, `@tsparticles/effect-bubble` |
| **Paths** | Control particle movement trajectories | `@tsparticles/path-perlin-noise`, `@tsparticles/path-svg` |
| **Plugins** | Everything else — emitters, absorbers, themes, sounds | `@tsparticles/plugin-emitters`, `@tsparticles/plugin-sounds` |

You load plugins by calling the `load*` function they export, passing the engine instance:

```javascript
import { tsParticles } from "@tsparticles/engine";
import { loadCircleShape } from "@tsparticles/shape-circle";
import { loadOpacityUpdater } from "@tsparticles/updater-opacity";

await loadCircleShape(tsParticles);
await loadOpacityUpdater(tsParticles);
```

<Tip>
  For most projects, `@tsparticles/slim` or the `tsparticles` full bundle is the right starting point. Use `@tsparticles/basic` when you need the smallest possible bundle, and `@tsparticles/all` only when you need advanced path generators or export functionality.
</Tip>

## Migrating from particles.js

If you have an existing particles.js configuration, the `@tsparticles/particles.js` compatibility package reads particles.js config files without any changes. Swap the script tag and you're done.
